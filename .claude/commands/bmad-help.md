# BMAD Method Help

Welcome to BMAD (Breakthrough Method for Agile AI-Driven Development) v6.0.0-Beta.2!

## Getting Started Commands

| Command | Description |
|---------|-------------|
| `/bmad-brainstorm` | **START HERE** - Creative ideation and concept development |
| `/bmad-party` | Multi-agent collaborative discussion |

## Available Agents

Use these slash commands to activate specialized AI agents:

| Command | Agent | Description |
|---------|-------|-------------|
| `/bmad-pm` | Product Manager | Strategic product management and requirements |
| `/bmad-analyst` | Business Analyst | Requirements gathering and analysis |
| `/bmad-architect` | Solution Architect | System design and architecture |
| `/bmad-dev` | Developer | Implementation and coding |
| `/bmad-sm` | Scrum Master | Agile practices and team facilitation |
| `/bmad-tea` | Testing Architect | Quality assurance and test design |
| `/bmad-ux` | UX Designer | User experience and interface design |
| `/bmad-tech-writer` | Technical Writer | Documentation and communication |
| `/bmad-quick` | Quick Flow Solo Dev | Rapid prototyping and iteration |

## Workflows

Workflows are located in `_bmad/workflows/`. Key workflow categories:

1. **Analysis** (`1-analysis/`) - Initial project analysis
2. **Planning** (`2-plan-workflows/`) - Project planning and scoping
3. **Solutioning** (`3-solutioning/`) - Design and architecture
4. **Implementation** (`4-implementation/`) - Development and deployment

## Quick Start

1. **Start with `/bmad-brainstorm`** to develop your idea
2. Use `/bmad-pm` to define your product vision
3. Use `/bmad-analyst` to gather requirements
4. Design with `/bmad-architect`
5. Implement with `/bmad-dev`
6. Test with `/bmad-tea`
